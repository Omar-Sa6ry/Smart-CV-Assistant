# Smart CV Assistant - Backend API

## Project Overview
The Smart CV Assistant Backend is a robust, scalable API built to power an intelligent resume creation and analysis platform. It handles user authentication, structured CV data management, PDF generation, and seamless integration with a Python-based AI model for Applicant Tracking System (ATS) compatibility scoring and content analysis.

## System Architecture
This project is built using a Modular Monolith architecture, prioritizing clean separation of concerns while maintaining deployment simplicity.

## Core Tech Stack
* **Framework:** NestJS (Node.js)
* **API Layer:** GraphQL (Code-First Approach)
* **Database:** PostgreSQL
* **ORM:** Prisma
* **Authentication:** JWT (JSON Web Tokens) & Google OAuth 2.0
* **Message Queue / Background Jobs:** Redis & BullMQ (For AI Integration)
* **PDF Generation:** Puppeteer (Headless Chrome)
* **File Storage:** Cloudinary

## Project Modules Breakdown

The backend is divided into domain-driven modules. Here is the exact structure and what each module contains:

### 1. `AuthModule` (Authentication & Security)
Handles all user identity and access management.
* **Controllers/Resolvers:** Login, Register, Google Auth Callback.
* **Services:** Validating credentials, signing JWTs, managing Google OAuth flow.
* **Guards:** `JwtAuthGuard` (protecting private queries/mutations).
* **Strategies:** `JwtStrategy`, `GoogleStrategy` (Passport.js).

### 2. `UsersModule` (User Management)
Manages the `User` entity generated by Prisma.
* **Resolvers:** Queries for fetching profile data, Mutations for updating user details (name, phone, role).
* **Services:** Interacting with the Prisma `user` model, tracking `lastLoginAt`.

### 3. `CvBuilderModule` (Core Data Management)
The largest module, responsible for the CRUD operations of the resume data.
* **Sub-Services:** * `CvService`: Manages the main `cvs` table (creating a CV, setting default).
  * `ExperienceService`: Manages the `experiences` table.
  * `EducationService`: Manages the `educations` table.
  * `SkillService`: Manages the `skills` table.
  * `ProjectService`, `CertificationService`, `LanguageService`.
* **Resolvers:** Provides the GraphQL endpoints for users to add, edit, or delete their CV sections. Ensures users can only modify data linked to their `userId`.

### 4. `AnalysisModule` (AI & Queue Integration)
Handles the asynchronous communication with the Python AI Model.
* **Producers (BullMQ):** Adds a job to the Redis queue when a user requests an ATS scan.
* **Consumers (Worker):** Listens to the queue, makes HTTP requests to the AI API, and processes the response.
* **Services:** Saves the incoming AI scores into `cv_analysis_base`, `cv_analysis_ats_details`, `cv_analysis_content_details`, and `cv_analysis_completeness_details`. Manages `analysis_history` to track user improvement over time.

### 5. `ExportModule` (PDF Generation & Storage)
Transforms structured database records into downloadable PDF files.
* **Services:** * `PdfGeneratorService`: Injects CV data into an HTML template and uses Puppeteer to render it as a high-quality PDF.
  * `CloudinaryService`: Uploads the generated PDF buffer to Cloudinary and saves the secure URL in the `cv_files` table.
* **Resolvers:** Returns the Cloudinary download URL to the client.

### 6. `DictionaryModule` (Auto-Suggestions)
Manages the static/reference data.
* **Services:** Interacts with the `skill_keywords` table to provide auto-complete suggestions for technical and soft skills while the user is typing in the builder.
